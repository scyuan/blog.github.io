失败的原因

浏览器是单线程的，UI 引擎和 JS 引擎互斥，也就是说，在循环运行的过程中， JS 引擎在执行循环操作时，无法调用UI引擎。所以涉及到重新渲染页面的操作不会立即执行，而是会排到任务队列中，等循环结束（JS 引擎空闲）时再执行渲染。所以能看到的效果是，页面假死，到最后突然变成排序结束的样子。

总结

这个实验前前后后大概花了10个小时左右，初期实现冒泡、选择、插入、快速排序。熟悉了一下这几个算法的思想：

  • 冒泡：每循环一次都将最大（小）的移动到了最后。每次都是相邻两个元素比较，整个过程就像水里冒泡一样；

  • 选择：和冒泡排序类似。但是比较的策略不一样。选择排序首先将第一个元素作为最大（或最小）的元素，然后循环，找到一个比这个更大（或者更小的）的值，然后替换，依次循环。

  • 插入排序：插入排序就相当于无序的扑克牌中一次抽出一张，插入到桌面上已经排好的序的列表中。比上面二者速度都快

  • 快速排序：也是以前最难理解的。其实只要理解分治策略即可，将问题分成小问题去解决。

然后再可视化的过程中，除了包括上面的那个问题外，还有熟悉了Jquery对DOM的操作：insertBefore和insertAfter

对初中级前端开发者来说，算法可能真正用到的地方很少，但是很重要的是这些算法的思想，我们去解决问题的思想有一些帮助吧。
