记第二次面试

发表于 2018-04-28   |   更新于 2018-05-04 | 分类于 面试 | | 热度: ℃
记第二次面试

毕业之后一年不到，前两天进行了两次面试，还有一次电话面试，总的来说，技术方面好像都没有太大的问题。第一次面试，应该说是电话面试，面的技术，问了一些技术方面的问题，好像不是很难，感觉也挺轻松的，所以HR邀请我第二次面试。

正题开始

开始自我介绍了一下，然后开始问我再公司了做了哪些项目，然后开始问细节，比如移动OA这个项目，负责哪些模块，做了一些什么工作，因为这个项目确实我做的少，所以答起来有点模模糊糊的，但是好歹都能回答上来。重点来了，面试官突然问我，这块的前端业务是怎样的？

WTF???前端还有业务吗？？我当时根本不知道前端业务是什么，后来他解释虽然是前端，但是也要关注业务逻辑，只有关注了业务逻辑，你才能真正知道用户真正是需要一个什么产品，这样你才能发现某个页面的不足，这样可以向上面提出改进。大概是这样的，就是一个前端的业务方面的能力，最后那个面试官跟我说我这方面能力不足，技术都没什么大问题（言外之意：技术都好实现）。想起了知乎上说的一段话：

技术热情，对技术问题深究的态度，善于总结，喜欢思考解决问题的各种方法和思路，能够自己快速解决各种技术问题。很宽泛的，但是我却认为是很重要的，我非常看重这一条，甚至超过对你的技术和经验的考察。我面试的时候一般不会问具体的技术细节，例如闭包是什么，写个排序算法，写个三栏布局什么的。其实如果你符合了上面说的那一条，这些问题对你来说只是随手google一下就能掌握的东西，问了意义并不会太大。创业公司，有时候技术真的没法搞的太高大上，我们需要的不是专家，而是能够从0开始，自我成长，自我约束，认同团队理念，懂得总结分享的人。然后我们尽力聚集越来越多拥有相同价值观的人，当然我们也会偶尔引入几个专家级别的开发，用外力来促进大家的成长，但是更多还是靠自我的驱动，我们提供的是一个氛围和环境。

后话

所以我之前都是陷入了一个误区，一直认为技术技术很重要，但其实技术都很好实现的啊，还是主要看你的思维，你能不能解决某个问题，你是怎样解决的，你是怎样思考问题的。

PS:第三次面试

第三次面试依然失败了，总结这三次面试，虽然一些JS、html、css基础我能答上来，这是没问题的，所以总是自我感觉良好，那么为什么总是面试不合格呢？显然，要么是真的不想要你，要么就是面试官觉得你不合适，因为如果你是面试官的话，你问的一些问题他有些回答不上来，那么就会觉得这个人不合适。所以虽然你觉得感觉良好，但是面试官的有些问题你答不上来，那就会被面试官pass。

首先面试官开头都会问为什么选择从上一家公司离职（貌似都问过了），这个问题该怎么回答？我一般都是说发展前景不好，项目不多，想去一家新公司，这样的回答是不是不完美？

然后会问一下技术方面的问题，技术方面的问题又会分为两部分，一部分是前端基础方面的问题，另一部分则是问一些项目上的事情。个人觉得前端基础方面的问题我是没多大问题的（当然还需要去熟悉，包括ES6），然后就是项目上的问题，做过哪些项目，会问这个项目是用来干嘛的，遇到过哪些比较难的问题，你的怎么解决的（这方面的问题比较重要？）。

项目上的问题可能自己需要提前去准备，这方面的问题我可能回答的并不好，

虽然面试淘汰了，但给我的成长还是挺多的，继续加油吧！
