使用git协同开发

发表于 2018-05-24 | 分类于 git |

前言

在项目中需要使用git来协同开发，在分支上push以及pull代码。

在项目开发过程中，慢慢在学习新的命令，对以前使用过的一些命令也更加熟悉和了解其原理

Git的原理

这部分下次再总结

Git 提交第一个项目

  • git init

初始化一个项目，生成一个本地文件仓库

  • git add .

将修改的文件添加到本地仓库

  • git commit -m “commit info”

提交至本地仓库

  • git remote add origin https://xxx/xxx.git

与远程仓库建立关联

  • git push -u origin master

提交至远程仓库

从远程仓库拉取项目

  • git clone https://xxx/xxx.git

一些其他比较常用的

  • git status

查看当前修改状态

  • git branch

查看本地分支

  • git branch -r

查看远程分支

  • git remote update

更新远程分支目录

  • git checkout .

删除所有本地所有已修改文件部分，回退到未修改状态

  • git checkout filepath

删除当前文件修改部分，回到未修改状态

git checkout new_branch
