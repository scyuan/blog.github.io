使用git协同开发

发表于 2018-05-24   |   更新于 2018-06-29 | | 热度: ℃

前言

在项目中需要使用git来协同开发，在分支上push以及pull代码。

在项目开发过程中，慢慢在学习新的命令，对以前使用过的一些命令也更加熟悉和了解其原理

Git的原理

这部分下次再总结

Git 提交第一个项目

  • git init

初始化一个项目，生成一个本地文件仓库

  • git add .

将修改的文件添加到本地仓库

  • git commit -m “commit info”

提交至本地仓库

  • git remote add origin https://xxx/xxx.git

与远程仓库建立关联

  • git push -u origin master

提交至远程仓库

从远程仓库拉取项目

  • git clone https://xxx/xxx.git

一些其他比较常用的

  • git status

查看当前修改状态

  • git branch

查看本地分支

  • git branch -r

查看远程分支

  • git remote update

更新远程分支目录

  • git checkout .

删除所有本地所有已修改文件部分，回退到未修改状态

  • git checkout filepath

删除当前文件修改部分，回到未修改状态

git checkout new_branch

新建一个分支

下面是爬过的坑，可以不用看，都是很弱智的。。。

修改代码之后第一次pull

报错：Your local changes to the following files would be overwritten by merge:
extends/integralshop/controller/mobile/static.php
Please commit your changes or stash them before you merge.

意思是：合并之后你本地修改的文件会被重写覆盖，请提交你的修改或者stash他们，在合并之前

（有个问题，就是git pull 的操作会拉最新的代码下来，然后再自动merge。如果没有merge，你本地的代码就没有更新）

上面报错，于是先git add . 然后再git commit -m “info”

这个时候本地仓库已经提交

再执行git pull

（我理解的）大致的意思是代码有冲突(解决冲突再commit)。(static.php)。于是我打开static.php将多出来乱码删除，再次git pull

再次出错

出错：Pulling is not possible because you have unmerged files.pull代码出错，因为你还有没有合并的文件。

网上的解决方案：https://blog.csdn.net/xinguan1267/article/details/39028789

1、git add -u

2、git commit -m “”

3、再次git pull
